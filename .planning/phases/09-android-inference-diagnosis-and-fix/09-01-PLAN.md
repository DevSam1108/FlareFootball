<objective>
Apply the two code changes needed before a physical-device run can confirm the Android YOLO callback pipeline.

Change 1 (DIAG-01 / DIAG-04): Add `aaptOptions { noCompress 'tflite' }` to `android/app/build.gradle`. This is the root-cause fix. Android's AAPT compresses all assets by default. A compressed `.tflite` file cannot be memory-mapped by `FileUtil.loadMappedFile()`. Without this block the interpreter constructor throws silently, `predictor` stays null, and `onResult` never fires.

Change 2 (DIAG-02 / DIAG-03): Add `log()` diagnostic calls inside the `onResult` callback in `live_object_detection_screen.dart`. These make it possible to confirm in the VS Code debug console that (a) `onResult` fires at all, and (b) `className` values match the custom model (`Soccer ball`, `ball`) rather than COCO fallback strings (`sports ball`).

Purpose: Without Change 1 the Android inference pipeline will remain silent regardless of other changes. Without Change 2 there is no observable evidence in the Flutter debug console â€” the developer would need to rely entirely on logcat.

Output: Modified `build.gradle` and `live_object_detection_screen.dart`, ready for `flutter clean && flutter pub get && flutter run`.
</objective>
