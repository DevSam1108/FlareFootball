<feature>
  <name>BallKalmanFilter — 4-state linear Kalman filter</name>
  <files>lib/services/kalman_filter.dart, test/kalman_filter_test.dart</files>
  <behavior>
    The filter maintains a 4-element state vector: [px, py, vx, vy] where px/py are
    normalized [0,1] ball position and vx/vy are velocity in normalized units per frame.

    State transition model (constant velocity):
      px' = px + vx * dt
      py' = py + vy * dt
      vx' = vx
      vy' = vy

    Where dt = 1.0 (one frame interval). The filter uses dt=1.0 for frame-based
    prediction; callers can scale velocity by actual frame rate if needed.

    Measurement model: we observe [px, py] directly from YOLO detections.
    Measurement matrix H = [[1,0,0,0],[0,1,0,0]].

    Tuning parameters (constructor args with sensible defaults):
      - processNoise (Q diagonal): how much we expect the ball to deviate from
        constant velocity per frame. Default: position=0.0001, velocity=0.01.
        Position noise is low (ball doesn't teleport), velocity noise is higher
        (ball accelerates/decelerates during kicks).
      - measurementNoise (R diagonal): how noisy YOLO detections are.
        Default: 0.005 for both px and py (YOLO jitter is ~0.5% of frame).

    Public API:
      - BallKalmanFilter({processNoise, measurementNoise}) -- constructor
      - void predict() -- advance state by one frame using constant velocity model
      - void update(double px, double py) -- correct state with measurement
      - (double px, double py) get position -- current smoothed position
      - (double vx, double vy) get velocity -- current estimated velocity
      - bool get isInitialized -- true after first update
      - void reset() -- clear state back to uninitialized

    First measurement behavior: On the very first update() call, the filter
    initializes state directly from the measurement (no prediction bias). Velocity
    starts at (0, 0). Covariance is initialized to moderate uncertainty.

    Cases:
      predict() when uninitialized -> no-op (returns without error)
      update(0.5, 0.5) first call -> position = (0.5, 0.5), velocity = (0, 0)
      predict() after update(0.5, 0.5) with velocity (0.1, 0) -> position ~ (0.6, 0.5)
      5 consecutive predict() calls -> position advances 5 frames along velocity
      update() after predict() -> position corrects toward measurement
      reset() -> isInitialized = false, position/velocity throw or return zero
  </behavior>
  <implementation>
    Implement as a pure Dart class with no Flutter dependencies. Use flat arrays
    (List<double>) for the 4x4 covariance matrix P, 4x4 state transition F,
    2x4 measurement matrix H, 4x4 process noise Q, and 2x2 measurement noise R.

    Matrix operations needed (all small, inline):
    - 4x4 * 4x1 multiply (state prediction)
    - 4x4 * 4x4 multiply (covariance prediction)
    - 4x4 transpose (for P update)
    - 2x2 inverse (for Kalman gain -- use direct formula for 2x2)
    - 4x2 * 2x1 multiply (Kalman gain * innovation)

    Do NOT use a general matrix library. The matrices are small and fixed-size.
    Inline the operations for clarity and performance, similar to how
    HomographyTransform implements its own linear algebra.

    The class should follow the same documentation style as HomographyTransform
    and BallTracker -- detailed doc comments on the class and public methods.
  </implementation>
</feature>
