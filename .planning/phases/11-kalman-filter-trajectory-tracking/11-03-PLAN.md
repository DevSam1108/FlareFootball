<feature>
  <name>TrajectoryExtrapolator — parabolic extrapolation to target plane</name>
  <files>lib/services/trajectory_extrapolator.dart, test/trajectory_extrapolator_test.dart</files>
  <behavior>
    The extrapolator takes the current ball state (position + velocity from the Kalman
    filter) and a calibrated TargetZoneMapper, and predicts where the ball's trajectory
    will intersect the target area.

    **Coordinate system:** All positions and velocities are in normalized camera space
    [0,1]. The target area is defined by the calibrated homography. The extrapolator
    works in camera-normalized space and uses the homography to test whether predicted
    points fall within the target.

    **Trajectory model:**
    In camera-normalized coordinates, the ball follows approximately:
      x(t) = px + vx * t
      y(t) = py + vy * t + 0.5 * g * t^2

    Where t is in frame units and g is a gravity constant in normalized units per
    frame^2. Gravity pulls downward in camera space (positive y direction in most
    orientations). The gravity constant is a tunable parameter -- default 0.001
    (subtle arc over ~30 frames). This is an approximation since camera perspective
    distorts the true parabola, but it's sufficient for POC-level prediction.

    **Algorithm:**
    1. Starting from current (px, py) with velocity (vx, vy), step forward in
       time increments of 1 frame.
    2. At each step, compute the predicted (x, y) using the parabolic model.
    3. Transform the predicted point through the homography to check if it lands
       inside the target area (0 <= tx <= 1 and 0 <= ty <= 1 in target space).
    4. If yes, return an ExtrapolationResult with:
       - cameraPoint: the predicted intersection in camera-normalized coords
       - targetPoint: the intersection in target-space coords
       - zone: the zone number (1-9) from TargetZoneMapper.pointToZone()
       - framesAhead: how many frames into the future the intersection occurs
    5. If the predicted point exits the camera frame (x < -0.1 or x > 1.1 or
       y < -0.1 or y > 1.1) without hitting the target, return null (miss).
    6. Maximum extrapolation horizon: 60 frames (~2 seconds at 30fps). If no
       intersection within 60 frames, return null.

    **Public API:**
      - TrajectoryExtrapolator({double gravity = 0.001, int maxFrames = 60})
      - ExtrapolationResult? extrapolate({
          required Offset position,
          required Offset velocity,
          required TargetZoneMapper zoneMapper,
        })

    **ExtrapolationResult** is a simple immutable class with fields:
      - Offset cameraPoint (normalized camera coords of intersection)
      - Offset targetPoint (target-space coords of intersection)
      - int zone (1-9)
      - int framesAhead

    Cases:
      Ball moving toward target, hits zone 5 -> ExtrapolationResult(zone: 5, ...)
      Ball moving away from target -> null (exits frame boundary)
      Ball moving parallel, never reaches target -> null (max frames exceeded)
      Ball at rest (velocity ~0) -> null (max frames exceeded, no movement)
      Velocity too small (< 0.001 magnitude) -> early return null (no meaningful trajectory)
  </behavior>
  <implementation>
    Create `lib/services/trajectory_extrapolator.dart` as a pure Dart class.
    Create `ExtrapolationResult` as an immutable data class in the same file.

    The extrapolate() method iterates frame-by-frame. For each frame t (1..maxFrames):
      x = position.dx + velocity.dx * t
      y = position.dy + velocity.dy * t + 0.5 * gravity * t * t
      Check bounds, check homography transform.

    Add a velocity magnitude threshold check at the start: if velocity magnitude
    < 0.001, return null immediately (ball is essentially stationary).

    For the test file, use the identity homography (unit square to unit square) for
    simple cases, and a known non-identity homography for perspective cases.
  </implementation>
</feature>
