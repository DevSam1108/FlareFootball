  <name>Task 1: Integrate Kalman filter into BallTracker and expose velocity API</name>
  <files>
    lib/services/ball_tracker.dart
    lib/models/tracked_position.dart
  </files>
  <action>
    **TrackedPosition changes (`lib/models/tracked_position.dart`):**
    Add optional `vx` and `vy` fields (double?, default null) to TrackedPosition.
    Add an `isPredicted` bool field (default false) to distinguish Kalman-predicted
    positions from measurement-backed positions. These are additive — the existing
    constructor contract is preserved (all new fields are optional with defaults).

    **BallTracker changes (`lib/services/ball_tracker.dart`):**

    1. Add a `BallKalmanFilter _kalman` field, initialized in the constructor.

    2. Modify `update(Offset normalizedCenter)`:
       - Call `_kalman.predict()` then `_kalman.update(dx, dy)` with the raw YOLO position.
       - Read the smoothed position from `_kalman.position` and use THAT for the
         trail entry instead of the raw `normalizedCenter`.
       - Read velocity from `_kalman.velocity` and store it in the TrackedPosition.
       - Keep the existing min-distance dedup logic, but apply it to the SMOOTHED
         position (compare Kalman output vs last trail entry, not raw input).
       - Keep the existing `_consecutiveMissedFrames = 0` reset.

    3. Modify `markOccluded()`:
       - Call `_kalman.predict()` on each missed frame.
       - For the first 5 missed frames (`_consecutiveMissedFrames <= 5`), instead
         of inserting an occlusion sentinel, insert a PREDICTED position from
         `_kalman.position` with `isPredicted: true`. This continues the trail
         through brief occlusions.
       - After 5 missed frames, revert to the existing sentinel behavior (insert
         occlusion sentinel, stop predicting).
       - Keep the existing `autoResetThreshold` (30 frames) behavior.
       - Add a `static const int predictionHorizon = 5` constant for the 5-frame limit.

    4. Add a `Offset? get velocity` getter that returns `Offset(vx, vy)` from the
       Kalman filter, or null if the filter is not initialized. This is the TRAJ-03
       API that downstream consumers (Phase 12 impact detection) will use.

    5. Add a `Offset? get smoothedPosition` getter that returns `Offset(px, py)` from
       the Kalman filter's current smoothed position, or null if the filter is not
       initialized. This is needed by Plan 11-03 so the trajectory extrapolator
       receives the Kalman-smoothed position rather than the raw YOLO detection.

    6. Modify `reset()` to also call `_kalman.reset()`.

    **Key constraint:** The TrailOverlay reads `TrackedPosition.isOccluded` to decide
    whether to break the polyline. Predicted positions during occlusion must have
    `isOccluded: false` so the line draws through them. Only sentinels inserted AFTER
    the 5-frame prediction horizon should have `isOccluded: true`.

    **Key constraint:** Do NOT change the TrailOverlay code. The existing painter
    already handles any TrackedPosition with isOccluded=false as a normal point.
    Predicted positions will render identically to detected positions.
  </action>
  <verify>

    `flutter test` -- all existing tests still pass (TrackedPosition constructor is backward-compatible).
  </verify>
