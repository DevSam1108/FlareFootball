<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify debug dot accuracy on both devices</name>
  <what-built>
Debug dot overlay rendering a red dot centered on the detected ball, with native YOLO bounding boxes suppressed, on both YOLO and SSD detection pipelines. Mounted guards protect against setState-after-dispose on navigation.
  </what-built>
  <how-to-verify>
**Test 1 — YOLO path (landscape mode):**
1. Launch the app with `DETECTOR_BACKEND=yolo` on each device
2. Navigate to Live Object Detection screen
3. Point the camera at a soccer ball on a flat surface
4. Verify: A red dot with white outline appears centered on the ball (not offset to a corner or edge)
5. Move the ball slowly — the dot should track it smoothly without visible lag
6. Verify: No native bounding boxes are visible (OVLY-03) — only the red dot
7. If diagnostic text is shown at bottom-left, check coordinates are in 0.0-1.0 range and move sensibly
8. Navigate back (pop the screen) and return — no crash or freeze

**Test 2 — SSD path (portrait mode):**
1. Relaunch with `DETECTOR_BACKEND=tflite` (or no flag — tflite is default)
2. Navigate to Live Object Detection screen
3. Point the camera at a ball
4. Verify: Red debug dot appears centered on the ball
5. Note: SSD bounding boxes (`BoxWidget`) will still be visible alongside the debug dot — this is expected for the SSD path (OVLY-03 only applies to YOLO native boxes)

**Test 3 — Navigation safety (either backend):**
1. On the detection screen, rapidly navigate back while detection is running
2. Check the debug console for `setState() called after dispose()` errors
3. Expected: No such errors (OVLY-04)

**Known issue to watch for (Galaxy A32):**
GitHub issue #105 documents a potential coordinate offset on some Android devices. If the dot is consistently shifted (e.g., always 20% toward center), note the direction and magnitude of the offset. This does NOT block the phase — document it for correction.

**Report:**
- Does the dot center on the ball on iPhone 12? (OVLY-01/OVLY-02)
- Does the dot center on the ball on Galaxy A32? (OVLY-01/OVLY-02)
- Are native YOLO boxes gone? (OVLY-03)
- Any setState-after-dispose errors? (OVLY-04)
- If offset observed, describe direction and approximate magnitude
