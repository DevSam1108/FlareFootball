**Step 2: Create `lib/utils/yolo_coord_utils.dart`**

Extract the FILL_CENTER crop offset math from `DebugDotPainter.paint()` (in `debug_dot_overlay.dart`) into a shared static utility. This is the exact same formula verified on iPhone 12 in Phase 6 — do NOT modify the math, only extract it.

The class must:
- Have a single static method: `static Offset toCanvasPixel(Offset normalized, Size canvasSize, double cameraAspectRatio)`
- Import only `dart:ui` for `Offset` and `Size`
- Use a private constructor to prevent instantiation: `YoloCoordUtils._();`
- Include the doc comment explaining FILL_CENTER behavior

The crop math logic (copied exactly from DebugDotPainter):
```dart
final widgetAR = canvasSize.width / canvasSize.height;
double pixelX, pixelY;

if (widgetAR > cameraAspectRatio) {
  // Widget wider than camera -> scaled by width, height cropped.
  final scaledHeight = canvasSize.width / cameraAspectRatio;
  final cropY = (scaledHeight - canvasSize.height) / 2.0;
  pixelX = normalized.dx * canvasSize.width;
  pixelY = normalized.dy * scaledHeight - cropY;
} else {
  // Widget taller than camera -> scaled by height, width cropped.
  final scaledWidth = canvasSize.height * cameraAspectRatio;
  final cropX = (scaledWidth - canvasSize.width) / 2.0;
  pixelX = normalized.dx * scaledWidth - cropX;
  pixelY = normalized.dy * canvasSize.height;
}
return Offset(pixelX, pixelY);
```

Create `lib/services/ball_tracker.dart` — a plain Dart service class (no Flutter widget dependencies) that manages the ball position history.

**Imports:**
- `dart:collection` for `ListQueue`
- `package:flutter/painting.dart` for `Offset` (lightweight — no widget framework)
- `package:tensorflow_demo/models/tracked_position.dart` for `TrackedPosition`

**Fields:**
- `final Duration trailWindow` — configurable, defaults to `const Duration(seconds: 1, milliseconds: 500)` per TRAK-01
- `static const int autoResetThreshold = 30` — consecutive missed frames before auto-clear per TRAK-05
- `final _history = ListQueue<TrackedPosition>()` — O(1) add/remove bounded queue
- `int _consecutiveMissedFrames = 0` — counter for auto-reset

**Public API:**

1. `List<TrackedPosition> get trail` — returns `List.unmodifiable(_history)` so the painter cannot mutate the queue

2. `Offset? get lastKnownPosition` — iterates `_history` in reverse, returns `normalizedCenter` of the first non-occluded entry. Returns null if no non-occluded entry exists. This is used by TRAK-04 nearest-neighbor tiebreaker in the screen.

3. `void update(Offset normalizedCenter)` — called when a ball IS detected:
   - Reset `_consecutiveMissedFrames = 0`
   - `_history.addLast(TrackedPosition(normalizedCenter: normalizedCenter, timestamp: DateTime.now()))`
   - Call `_prune()`

4. `void markOccluded()` — called when NO ball is detected in a frame:
   - Increment `_consecutiveMissedFrames`
   - If `_consecutiveMissedFrames >= autoResetThreshold`, call `reset()` and return (TRAK-05)
   - Only insert a sentinel if `_history.isNotEmpty && !_history.last.isOccluded` — prevents sentinel stacking (TRAK-02)
   - Sentinel uses `normalizedCenter` from `_history.last.normalizedCenter` and `isOccluded: true`
   - Call `_prune()`

5. `void reset()` — clears `_history` and resets `_consecutiveMissedFrames = 0`

**Private:**

6. `void _prune()` — time-based eviction:
   - `final cutoff = DateTime.now().subtract(trailWindow)`
   - While `_history.isNotEmpty && _history.first.timestamp.isBefore(cutoff)`, call `_history.removeFirst()`
   - CRITICAL: Do NOT reset `_consecutiveMissedFrames` inside `_prune()` — that counter is only reset in `update()` (see research Pitfall 3)

**Constructor:** `BallTracker({this.trailWindow = const Duration(seconds: 1, milliseconds: 500)});`
