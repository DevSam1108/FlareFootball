```dart
// "Ball lost" badge — PLSH-01.
// IgnorePointer: must not consume touch events from YOLOView.
if (_tracker.isBallLost)
  IgnorePointer(
    child: Positioned(
      top: 12,
      right: 12,
      child: Container(
        padding: const EdgeInsets.symmetric(horizontal: 10, vertical: 6),
        decoration: BoxDecoration(
          color: Colors.red.withValues(alpha: 0.85),
          borderRadius: BorderRadius.circular(8),
        ),
        child: const Text(
          'Ball lost',
          style: TextStyle(
            color: Colors.white,
            fontSize: 12,
            fontWeight: FontWeight.w600,
          ),
        ),
      ),
    ),
  ),
```

Key placement rules:
- AFTER the backend label `Positioned` (last existing child) so it renders on top
- OUTSIDE the trail `RepaintBoundary` — it is a separate Stack sibling
- Wrapped in `IgnorePointer` (outside `Positioned`) — consistent with trail overlay pattern from Phase 7
- Position: `top: 12, right: 12` — the backend label occupies `top: 12, left: 12`
- Use `Colors.red.withValues(alpha: 0.85)` — NOT deprecated `withOpacity()`
- Use `const Text` and `const EdgeInsets` where possible

No additional `setState` needed — the badge piggybacks on the existing `setState` in the `onResult` callback which fires every YOLO frame. The `build` method reads `_tracker.isBallLost` on every rebuild.

Do NOT:
- Add a Timer, Stream, or animation controller for the badge
- Place the badge inside the trail RepaintBoundary
- Remove the IgnorePointer wrapper
- Modify the onResult callback or any other existing code
